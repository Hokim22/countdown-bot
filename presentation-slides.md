# プレゼンテーション用スライド構成

## スライド1: システム構成図
```
カウントダウンbotシステム構成

EventBridge → Lambda → DynamoDB
     ↓           ↓
   1時間毎     メイン処理
     
Lambda → Bedrock → Slack
   ↓        ↓        ↓
キャラ選択  AI生成  メッセージ投稿
```

## スライド2: 使用サービス一覧
```
【使用AWS サービス】

🕐 EventBridge    - スケジューラー
⚡ Lambda         - 処理エンジン  
💾 DynamoDB       - データベース
🤖 Bedrock        - AI文章生成
📱 Slack          - 通知先
```

## スライド3: EventBridge詳細
```
Amazon EventBridge
【役割】スケジューラー・トリガー

✅ 定期実行: rate(1 hour) で1時間に1回
✅ イベント駆動: 自動的にLambdaを起動
✅ 高信頼性: 99.99%の可用性

【選定理由】
・サーバーレスでスケジュール実行
・従量課金で低コスト（月額$0.1）
```

## スライド4: Lambda詳細
```
AWS Lambda
【役割】メイン処理エンジン

✅ ビジネスロジック: カウントダウン計算
✅ データ処理: DynamoDB読み取り
✅ 通知処理: Slackメッセージ投稿

【選定理由】
・サーバー管理不要
・実行時間分のみ課金（月額$1-2）
・自動スケーリング
```

## スライド5: DynamoDB詳細
```
Amazon DynamoDB
【役割】設定データベース

✅ データ保存: 試験情報の永続化
✅ 高速読み取り: ミリ秒レベルアクセス
✅ 構造化データ: JSON形式で柔軟管理

保存データ例:
{
  "examName": "社内勉強会での発表",
  "targetDate": "2025-10-14",
  "slackWebhookUrl": "https://hooks.slack.com/..."
}

【選定理由】
・NoSQLで柔軟なスキーマ
・サーバーレスで管理不要（月額$1）
```

## スライド6: Bedrock詳細
```
Amazon Bedrock
【役割】AI文章生成エンジン

✅ AIモデル: Claude 3 Haiku使用
✅ 文章生成: キャラクター別メッセージ
✅ プロンプト処理: 個性的な応援文作成

生成例:
「はぁ？まだスライド作ってないの？
あと3日しかないよ？💦 まずは
アウトライン作成から始めなよ...」

【選定理由】
・高品質な日本語生成
・キャラクター別個性表現（月額$2-5）
```

## スライド7: Slack連携詳細
```
Slack Webhook API
【役割】メッセージ配信

✅ メッセージ投稿: 指定チャンネルへ自動投稿
✅ Webhook連携: HTTPSで簡単連携
✅ Bot設定: カスタムユーザー名・アイコン

投稿内容:
📚 社内勉強会での発表 まであと 3日
🎭 今日のキャラ: 毒舌系フリーター
[AI生成応援メッセージ]

【選定理由】
・簡単なWebhook連携
・リアルタイム通知
・無料で利用可能
```

## スライド8: サービス間連携フロー
```
【処理フロー詳細】

1️⃣ EventBridge
   「1時間経ったよ！Lambda起動して！」

2️⃣ Lambda → DynamoDB  
   「今日処理する試験データをください」

3️⃣ Lambda → Bedrock
   「毒舌系フリーターで応援メッセージ作って」

4️⃣ Lambda → Slack
   「このメッセージをSlackに投稿して」
```

## スライド9: コスト内訳
```
【月額運用コスト】

EventBridge  │ $0.1  │ ルール実行回数
Lambda       │ $1-2  │ 実行時間・メモリ  
DynamoDB     │ $1    │ 読み書き・ストレージ
Bedrock      │ $2-5  │ AI生成トークン数
Slack        │ 無料   │ Webhook利用
─────────────┼───────┼─────────────────
合計         │ $4-8  │ 従量課金モデル
```

```
【なぜこの構成なのか】

🚀 サーバーレス統一
   ・管理負荷ゼロ
   ・自動スケーリング
   ・高可用性

💰 従量課金モデル  
   ・使った分だけ支払い
   ・個人利用に最適
   ・無駄なコストなし

🔗 AWS エコシステム
   ・サービス間連携が簡単
   ・統一された権限管理
   ・一元的な監視・ログ
```
